Все значимые изменения скилла фиксируются в этом файле.  
Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/).  
Версионирование: `MAJOR.MINOR.PATCH`

- **MAJOR** — несовместимые изменения (переименование файлов, смена формата страниц)

## [2.0.0] — 2026-06-25

### Добавлено

#### Tree-sitter парсер в `repo-map.py` (приоритетный)
- `repo-map.py` теперь использует tree-sitter как основной парсер (31+ язык: TypeScript, TSX, JavaScript, Rust, C#, Swift, Ruby, C, C++, Scala, PHP, Dart, Lua, и другие).
- Regex-парсеры (Python, Java, Go, Kotlin, XML) переведены в статус **fallback**: используются автоматически если tree-sitter не установлен или не поддерживает данный язык.
- Добавлен флаг `--no-ast` для принудительного использования только regex (совместимость, отладка).
- Добавлена переменная `_TREESITTER_AVAILABLE` — тихое автоопределение при импорте без исключений.
- `collect_files()` теперь собирает файлы всех поддерживаемых расширений (`_ALL_EXTENSIONS` = regex + tree-sitter-only), а не только те, для которых есть regex-парсер.
- Поле `**Парсер**` добавлено в генерируемый `repo-map.md` — показывает активный режим (`tree-sitter` или `regex`).
- Установка пакетов — опциональная зависимость: при отсутствии поведение идентично предыдущей версии.

#### Git post-commit хук (`wiki/scripts/install-hook.sh`)
- Новый скрипт установки git-хука (8 строк + интерактивное подтверждение при перезаписи).
- Хук запускает `repo-map.py --incremental` в фоне (`&`) после каждого `git commit`.
- Не блокирует коммит, не вызывает LLM — только структурная карта.
- `repo-map.md` всегда актуален к началу следующей сессии.
- Установка: `sh wiki/scripts/install-hook.sh` из корня проекта.

#### Теги уверенности (`**Уверенность**`)
- Новое обязательное поле во всех шаблонах страниц: `**Уверенность**: extracted | inferred | ambiguous`.
- `extracted` — факт взят напрямую из кода/конфига.
- `inferred` — агент предположил из контекста (назначение модуля, неявные связи).
- `ambiguous` — конфликт источников или неясность, требует проверки человеком.
- Добавлен раздел «Теги уверенности» в `SKILL.md` с таблицей и правилами применения.
- Теги добавлены в шаблоны: модуль, понятие, решение, план.
- Интеграция с `/wiki-lint`: страницы `inferred + draft` → 🟡 ВНИМАНИЕ.

#### Hub-stubs и orphan clusters в `/wiki-lint`
- 🟡 **Hub-stub**: страница с >5 входящих `[[wikilinks]]` и <300 символов контента. Алгоритм: граф по indegree + проверка длины контента.
- 🟡 **Orphan cluster**: группа страниц, связанных между собой, но без рёбер в/из основного графа. Алгоритм: связные компоненты без внешних рёбер.
- 🔵 **Уверенность отсутствует**: страницы без поля `**Уверенность**` — информационная пометка.
- Итоговая строка lint обновлена: `hub-stubs: N | orphan clusters: N`.

#### Q&A-archive (`**Тип**: q&a-archive`)
- Страницы, созданные через `/wiki-ask` → «Сохранить», получают два новых поля:
  - `**Уверенность**: inferred` (ответ основан на wiki, а не на прямом чтении кода)
  - `**Тип**: q&a-archive` (не первичный источник)
- При следующем `/wiki-ask` агент использует эти страницы как дополнительный контекст, но верифицирует по полю `Источники` перед доверием.

- `opencode/commands/wiki-lint.md`: добавлены hub-stubs, orphan clusters, `inferred + draft` проверки в 🟡; поле `**Уверенность**` в 🔵; обновлена итоговая строка.
- `README.md`: добавлены разделы — `install-hook.sh`, tree-sitter установка, таблица механизмов, инструкция обновления скилла.
- `TUTORIAL.md`: обновлены шаги установки (2.5 хук, 2.6 tree-sitter), раздел 4 (механизмы), приложения A–H, примеры lint-вывода.

### Не затронуто (фактчекинг)

Следующие предложения из анализа **не реализованы** как неактуальные или уже существующие:

| Предложение | Причина отклонения |
|---|---|
| Content-hash кэш | Уже реализован в `.source-hashes.json`. Перевод `.repo-map-cache.json` с mtime на хеш — микрооптимизация, не приоритет |
| LESSONS.md с time-decay | В Graphify v8 отсутствует (агент галлюцинировал версию 0.8.47). Самостоятельная разработка не оправдана |
| Progressive disclosure SKILL.md | Уже реализовано через slash-команды в `opencode/commands/` |
| Leiden кластеризация | Wiki — осознанная архитектура (ADR), статистические кластеры дадут ложные срабатывания |
| graph.html визуализация | Целевой пользователь — AI-агент, не человек |
| MCP сервер | Оправдан при 100+ страницах; преждевременно |
| `--watch` daemon | Противоречит модели взаимодействия скилла |

### Совместимость

**Обратная совместимость сохранена** для изменений 2–5. Существующие проекты продолжают работать без изменений.

Для изменения 1 (tree-sitter): если `pip install tree-sitter` не выполнялся — поведение идентично v1.x.

**Требует ручного действия**: добавить поле `**Уверенность**` на существующих страницах вики. `/wiki-lint` 🔵 покажет страницы без поля.
