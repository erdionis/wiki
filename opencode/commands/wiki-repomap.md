---
description: "Регенерация структурного скелета (`repo-map.md` + `file-page-map.json` шаблон)"
---

Сгенерируй или обнови структурную карту репозитория: $ARGUMENTS

Карта (repo-map) — низкоуровневый структурный скелет кодовой базы:
классы, интерфейсы, методы, функции. Дополняет вики-страницы modules/
конкретными сигнатурами. Генерируется скриптом, не LLM.

─────────────────────────────────────────
ШАГ 1 — ЗАПУСК СКРИПТА
─────────────────────────────────────────

Проверь доступность Python:
  python wiki/scripts/repo-map.py $ARGUMENTS

Если Python недоступен, используй bash-версию:
  bash wiki/scripts/repo-map.sh $ARGUMENTS

Скрипт сам определит шестерню по числу файлов:
  Gear 1 (< 30 файлов)  — полные сигнатуры методов
  Gear 2 (30–150 файлов) — классы + имена методов
  Gear 3 (> 150 файлов)  — структура папок + точки входа

Принудительно задать шестерню:
  python wiki/scripts/repo-map.py --gear 1
  python wiki/scripts/repo-map.py --gear 2

─────────────────────────────────────────
ШАГ 2 — ПОСЛЕ ГЕНЕРАЦИИ
─────────────────────────────────────────

После успешного запуска скрипта:
1. Прочитай сгенерированный wiki/architecture/repo-map.md
2. Найди в нём классы/модули, для которых в wiki/modules/ нет страниц
3. Выведи список: «Эти компоненты есть в repo-map, но не проиндексированы в вики:»
4. Предложи: /wiki-ingest <метка> для недостающих компонентов
5. Добавь запись в wiki/log.md об обновлении карты

─────────────────────────────────────────
ШАГ 3 — ИНТЕГРАЦИЯ В ВИКИ
─────────────────────────────────────────

Обнови wiki/index.md — убедись, что есть запись:
  [[architecture/repo-map]] — Структурная карта: классы, методы, функции

repo-map.md НЕ редактируется вручную — только через скрипт.
При каждом /wiki-update скрипт должен перезапускаться автоматически.

Если $ARGUMENTS не указан — запустить для всех меток src-* из wiki/sources.md.
