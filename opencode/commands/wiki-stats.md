Покажи статистику и состояние вики: $ARGUMENTS

Собери и выведи сводку по текущему состоянию базы знаний проекта.

══════════════════════════════════════════
СБОР ДАННЫХ
══════════════════════════════════════════

Прочитай wiki/ рекурсивно. Подсчитай:

1. ОБЪЁМ
   - Всего страниц (по разделам: modules/, concepts/, decisions/, env/, plans/, qa/, architecture/)
   - По жизненному циклу: active / draft / stale / archived (поле **Статус**)
   - Страниц с пометками [устарело] / [изменено:*] / [требует проверки]
   - Последняя запись в wiki/log.md (дата последнего обновления)

2. ПОКРЫТИЕ
   - Меток в wiki/sources.md: сколько всего
   - Меток с хотя бы одной wiki-страницей: сколько
   - Меток без ни одной страницы: список

3. СВЯЗНОСТЬ
   - Страниц без исходящих [[ссылок]]: сколько
   - Страниц без входящих [[ссылок]] (осиротевшие): сколько
   - Средняя плотность ссылок (ссылок / страница)

4. ИНФРАСТРУКТУРА
   - L1 hot cache: есть / нет, свежесть (часов с последнего обновления)
   - SHA-256 хеши: есть / нет (.source-hashes.json)
   - Дата последней генерации wiki/architecture/repo-map.md
   - Есть ли wiki/architecture/overview.md

══════════════════════════════════════════
ВЫВОД
══════════════════════════════════════════

Формат вывода:

  ┌─────────────────────────────────────────┐
  │  WIKI STATS — {PROJECT_NAME}            │
  │  {дата}                                 │
  ├─────────────────────────────────────────┤
  │  Страниц всего:     N                   │
  │    modules/         N                   │
  │    concepts/        N                   │
  │    decisions/       N                   │
  │    plans/           N                   │
  │    qa/              N                   │
  │    architecture/    N                   │
  ├─────────────────────────────────────────┤
  │  Жизненный цикл:                       │
  │    ✅ active:      N                   │
  │    📝 draft:       N                   │
  │    ⏳ stale:       N                   │
  │    🗄 archived:    N                   │
  ├─────────────────────────────────────────┤
  │  🔴 Устарело:       N страниц           │
  │  🟡 Изменено:       N страниц           │
  │  🟡 Нет источника:  N страниц           │
  ├─────────────────────────────────────────┤
  │  Покрытие источников: N/M меток         │
  │  Непокрытые метки: label-a, label-b     │
  ├─────────────────────────────────────────┤
  │  Осиротевших страниц: N                 │
  │  Без исходящих ссылок: N               │
  │  Плотность ссылок: X.X /стр             │
  ├─────────────────────────────────────────┤
  │  Hot cache:  свежий / устарел / нет     │
  │  SHA-256:    есть / нет                 │
  │  Repo-map:   актуален / устарел / нет   │
  │  Overview:   есть / нет                 │
  │  Обновлён:   ГГГГ-ММ-ДД               │
  └─────────────────────────────────────────┘

  ОЦЕНКА ЗДОРОВЬЯ:
  🟢 Хорошо   — покрытие > 80%, устаревших < 5%, осиротевших < 10%
  🟡 Внимание — покрытие 50–80% или устаревших 5–20%
  🔴 Требует работы — покрытие < 50% или устаревших > 20%

  РЕКОМЕНДАЦИИ (если есть проблемы):
  1. ...

Если $ARGUMENTS содержит "details" — дополнительно вывести
полные списки осиротевших страниц и непокрытых меток.
