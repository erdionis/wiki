Восстанови контекст и подготовься к работе: $ARGUMENTS

══════════════════════════════════════════
ВСЕГДА — восстановление контекста
══════════════════════════════════════════

1. Прочитай wiki/sources.md
2. Прочитай wiki/architecture/overview.md (если есть)
3. **L1 hot cache**: если существует `wiki/hot-context.md` и обновлён < 24ч — прочитай его
   **L2 index**: если hot-context устарел или отсутствует — прочитай `wiki/index.md`
4. Выведи краткую сводку:

   ПРОЕКТ: {название}
   Проиндексировано: {N страниц по разделам}
     active: N | draft: N | stale: N | archived: N
   Требуют внимания: {страницы со статусом stale или пометкой [устарело] / [изменено:]}
   Не проиндексировано: {метки из sources.md без wiki-страниц, если есть}

══════════════════════════════════════════
ЕСЛИ $ARGUMENTS указан — wiki-first протокол для задачи
══════════════════════════════════════════

После сводки выполнить разведку по задаче:

ФАЗА 1 — что уже известно из wiki:
- Найти ВСЕ страницы, связанные с задачей (modules/, concepts/, decisions/, env/)
- Прочитать каждую
- Вывести: что известно / какие пробелы остались

ФАЗА 2 — заполнение пробелов (если они есть):
- Для каждого пробела: найти в wiki/architecture/repo-map.md нужный файл
- Прочитать ТОЛЬКО этот файл
- Записать находку в wiki НЕМЕДЛЕННО (до продолжения)
- Правило: нашёл в коде → положил в wiki → второй раз не ищем

ФАЗА 3 — план выполнения:
- Предложить конкретные шаги на основе wiki
- Указать, какие wiki-страницы будут затронуты

ФАЗА 4 — ретроспектива (после выполнения задачи, по запросу):
  ЧТО ПОМОГЛО из wiki: ...
  ЧТО ДОБАВЛЕНО в wiki: ...
  ЧТО ЕЩЁ НУЖНО проиндексировать: ...

══════════════════════════════════════════

Примеры вызова:
  /wiki-start                              → только контекст сессии
  /wiki-start добавить рефанд в платёжный модуль   → контекст + разведка по задаче
